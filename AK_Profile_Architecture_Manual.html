AK Red-Team Patch Manual v1.2

主頁不是名片，
是冷流量的第二次審查頁。

受眾看完短影片後，不一定會立刻私訊。很多人會先點進你的主頁，判斷：你是誰、是否可信、是否和我有關、下一步要去哪裡。

本手冊是「主頁防漏」補強模組。目標不是把主頁變漂亮，而是避免受眾點進主頁後，因為看不懂你、找不到證據或不知道下一步而離開。

0. 新手名詞對照表

先看中文意思，再看英文術語。英文只是方便對照平台介面。

個人簡介（Bio）主頁上方那幾行文字。它要快速說清楚：你幫誰、解決什麼、下一步去哪裡。
置頂內容（Pin / Pin Reels / Pin Posts）固定在主頁最上方的內容。不是放最紅的，而是放最能讓陌生人理解你、信任你、知道下一步的內容。
精選限動（Highlight）IG 主頁上可以長期保存的限動分類。用來保存常見問題、案例、流程、工具入口與客戶回饋。
個人檔案連結（Link in Bio）主頁可點擊的連結。它不是連結倉庫，而是把人導向最重要的下一步，例如檢查表、LINE、工具頁或預約表單。
私訊觸發句（DM Trigger）引導對方私訊你的句子或關鍵字，例如「回覆『路線』我給你 AI 學習檢查表」。重點是自然、低壓、不要互動誘餌。
主頁防漏意思是：受眾點進主頁後，不要因為看不懂你是誰、沒有信任證據、找不到下一步而離開。

1. 主頁防漏是什麼？

一句話定義

主頁防漏，就是把你的個人主頁設計成「內容流量的中繼站」，用個人簡介、置頂內容、精選限動、連結與私訊觸發句，把看過內容的人導向低壓力下一步。

紅隊指出的問題

如果受眾看完影片後點進主頁，卻看不懂你是誰、你能幫什麼、為什麼可信、下一步去哪裡，前端所有流量都會在主頁漏掉。

簡單公式

主頁防漏 = 個人簡介清楚 × 置頂內容有分工 × 精選限動能補信任 × 連結 / 私訊路徑清楚 × 數據能回收檢查

這是 AK 的實務框架，不是平台官方規則，也不是成交保證。

2. 主頁不是什麼？

不要把主頁做成

  • 自我介紹頁
  • 履歷牆
  • 所有連結的倉庫
  • 漂亮排版展示
  • 平台推薦公式
  • 成交保證頁

主頁真正要回答

  • 你幫誰？
  • 你解決什麼問題？
  • 你為什麼可信？
  • 有哪些證據？
  • 我下一步去哪裡？

3. 主頁在整體流程中的位置

短影片 / 貼文
→
主頁
→
個人簡介 / 置頂 / 精選限動
→
私訊 / 連結 / 產品入口

主頁是「中繼站」。它不是最前端的內容，也不是最後的成交頁，而是讓陌生人確認：這個人是否值得我繼續看、私訊、點連結或領取資源。

4. 主頁防漏五層架構

1 個人簡介清楚

3–4 行說清楚你幫誰、解決什麼、用什麼方法、下一步去哪裡。

2 置頂內容有分工

三則置頂內容分別回答：你是誰、你能幫什麼、下一步如何開始。

3 精選限動補信任

把高價值限動保存成常見問題、案例、流程、工具入口、回饋。

4 連結 / 私訊路徑清楚

讓訪客知道低壓力下一步，不要把主頁變成連結倉庫。

5 數據回收檢查

用主頁瀏覽、連結點擊、私訊數、有效私訊數檢查漏點。

注意

主頁做好不保證成交，只能降低「看完內容但不知道下一步」的流失。

5. 個人簡介（Bio）怎麼寫？

個人簡介不是人生介紹，而是把你的 Strategy 壓縮成幾行。

建議格式

我幫助 [具體受眾] 解決 [具體痛點 / 結果缺口] 用 [方法 / 系統 / 工具 / 經驗] 下一步：[低壓力行動]

範例

幫 AI 新手建立工具學習路線 從工具清單 → 工作流 → 作品集 用 AK 系統把內容變成可執行流程 👇 領取 AI 學習路線檢查表

常見錯誤

  • 只寫頭銜，例如「AI 顧問 / 內容創作者」。
  • 寫太多人生理念，沒有說你幫誰。
  • 沒有具體問題。
  • 沒有下一步。
  • 影片叫人私訊，主頁卻叫人點別的東西。

6. 置頂內容三件套

置頂內容不是放最紅的，而是放最能承接陌生人的內容。

位置任務回答訪客的問題
置頂 1｜你是誰 / 為什麼讓人知道你是誰、為什麼做這件事這個人和我有關嗎？
置頂 2｜問題 / 方法說明你解決什麼問題、用什麼方法他能幫我解決什麼？
置頂 3｜證據 / 下一步提供案例、成果、資源或私訊入口我下一步去哪裡？

教學型帳號替代版

從這裡開始 最核心方法 免費資源 / 私訊入口

7. 精選限動怎麼規劃？

精選限動不是日常雜物櫃，而是長期信任證據庫。

分類用途
從這裡開始讓新訪客知道第一步怎麼看。
常見問題回答受眾最常問的疑慮。
案例 / 成果保存可驗證的成果與案例條件。
流程展示你如何工作、如何帶人完成。
免費資源放檢查表、模板、工具入口。
幕後讓人看到真實過程與日常。
回饋保存學員、客戶或使用者回饋。

簡單原則

保留 5–7 個核心分類即可。太多會像雜物櫃，太少又無法回答疑慮。重點不是封面漂亮，而是內容能補信任。

8. 個人檔案連結 / 私訊路徑

主頁上的連結與私訊觸發句，要把人導向低壓力下一步。

可用入口

  • 私訊關鍵字
  • 免費檢查表
  • 模板
  • LINE / 社群
  • 預約諮詢
  • 低價產品
  • 工具頁
  • 等待名單

五個原則

  • 主入口只能有一個。
  • 次入口可以有，但不能搶焦點。
  • 入口要和 Strategy / Conversion Path 對齊。
  • 不要讓冷流量看到太多選項。
  • 個人檔案連結不是連結倉庫，是轉化分流器。

9. 怎麼看主頁有沒有漏？

看到的數據現象可能代表什麼
主頁瀏覽高、私訊低個人簡介、置頂內容或行動指令承接弱。
主頁瀏覽高、連結點擊低連結入口不清楚，或吸引力不足。
連結點擊高、私訊低落地頁、表單或免費資源承接斷掉。
私訊多但品質差行動指令吸到錯的人，或缺少篩選問題。
主頁瀏覽低短影片或腳本沒有讓人想進一步了解你。

真正要看的是這條鏈

主頁瀏覽 → 連結點擊 / 私訊 → 有效私訊 → 轉化

10. 主頁檢查表

個人簡介檢查

  • 是否明確說出服務誰？
  • 是否說出解決什麼問題？
  • 是否有方法 / 系統 / 角度？
  • 是否有單一下一步？
  • 是否和 Strategy 一致？

置頂內容檢查

  • 第 1 則是否回答你是誰？
  • 第 2 則是否回答你怎麼解？
  • 第 3 則是否回答證據 / 下一步？
  • 三則是否互補而非重複？
  • 是否和轉化路徑一致？

精選限動檢查

  • 是否有「從這裡開始」？
  • 是否有常見問題？
  • 是否有案例或成果？
  • 是否有流程？
  • 是否刪除過期分類？

連結 / 私訊檢查

  • 是否只有一個主入口？
  • 主入口是否和內容行動指令一致？
  • 是否有低壓力入口？
  • 是否避免過多選項？
  • 私訊第一問是否和入口一致？

11. 常見失敗診斷

失敗症狀根因修正
主頁瀏覽高但沒私訊很多人點進來但不行動個人簡介或置頂內容沒有下一步重寫個人簡介與置頂第 3 則。
主頁瀏覽高但連結點擊低沒人點連結入口不明或選項太多保留單一主入口。
私訊多但品質差來的人不適合個人簡介或行動指令沒有篩選加入 Not For 或篩選問題。
置頂觀看高但不轉化置頂很熱但無承接置頂只娛樂，沒有信任矩陣重分配三則置頂任務。
精選限動很多但無用看起來像雜物櫃沒有信任證據庫架構重整成 5–7 個核心分類。
主頁和內容斷裂點進來像不同人定位、行動指令、限動不一致統一主線與入口。

13. 不可以誇大的說法

主頁做好就一定成交

錯。主頁只能降低承接斷點，不保證成交。

個人簡介寫對就一定漲粉

錯。個人簡介是承接工具，不是流量保證。

置頂三件套會平台加權

錯。這是信任承接設計，不是平台排名因子。

連結越多越完整

錯。過多選項會提高決策摩擦。

14. 建議放在整套流程的位置

主頁防漏是必要補強模組，不是新主線章節。建議位置：

Strategy → 主頁防漏 → Signal → Hook → Script → CTA → Stories → Conversion → Algorithm → Performance → Execution
