AK Red-Team Patch Manual v1.1

Profile 不是名片，
是冷流量的二次審查頁。

受眾看完 Reel / Short 後，不一定直接 DM。很多人會先點進主頁判斷：你是誰、是否可信、是否和我有關、下一步要去哪裡。

本模組是紅隊補強，不是新主線章節。它補上 Reel / Short → Profile → 個人簡介 / 置頂內容 / 精選限動 / 個人檔案連結 / 私訊觸發句 的主頁防漏層。

0. 新手名詞對照表

本手冊採「中文意思優先，英文術語放括號」。先理解功能，再記英文。

個人簡介（Bio） 主頁上方那幾行文字。它不是自我介紹，而是要讓新訪客快速知道：你幫誰、解決什麼、下一步去哪裡。
置頂內容（Pin / Pin Reels / Pin Posts） 固定在主頁最上方的 3 則內容。不是放最紅的，而是放最能讓陌生人理解你、信任你、知道下一步的內容。
精選限動（Highlight） IG 主頁上可以長期保存的限動分類。用來保存 FAQ、案例、流程、學員回饋、免費資源，不讓高價值限動 24 小時後消失。
個人檔案連結（Link in Bio） 主頁可點擊的連結。它不是連結倉庫，而是把人導向最重要的下一步，例如檢查表、LINE、工具頁或預約表單。
私訊觸發句（DM Trigger） 引導對方私訊你的句子或關鍵字。例如：「回覆『路線』我給你 AI 學習檢查表」。重點是自然、低壓、不要互動誘餌。
主頁防漏 意思是：受眾點進主頁後，不要因為看不懂你是誰、沒有信任證據、找不到下一步而離開。

1. Profile Architecture 是什麼？

一句話定義

Profile Architecture 是把個人主頁設計成內容流量的轉化中繼站，用 Bio、置頂內容、Highlight、連結與 DM Trigger 承接冷流量並導向低摩擦下一步。

紅隊問題

如果受眾看完影片後進主頁，卻看不懂你是誰、能幫什麼、下一步去哪裡，前端所有流量都會在主頁漏掉。

工具化公式

Profile Architecture = 個人簡介清晰度（Bio Clarity） × 置頂內容信任矩陣（Pin Trust Matrix） × 精選限動證據庫（Highlight Proof Bank） × 個人檔案連結 / 私訊路徑（Link / DM Path） × 成效回收檢查（Performance Feedback）

此框架屬 practical_inference / first_party_framework，不是平台官方推薦規則或成交保證。

2. Profile 不是什麼

不要把主頁做成

  • 自我介紹頁
  • 履歷牆
  • 所有連結的倉庫
  • 漂亮排版展示
  • 平台推薦公式
  • 成交保證頁

它真正要回答

  • 你幫誰？
  • 你解決什麼問題？
  • 你為什麼可信？
  • 有哪些證據？
  • 我下一步去哪裡？

4. Profile 五層架構

1 個人簡介清晰度（Bio Clarity）

3–4 行回答你幫誰、解決什麼、用什麼方法、下一步去哪裡。

2 置頂內容信任矩陣（Pin Trust Matrix）

三支置頂內容分工：你是誰、你能幫什麼、下一步如何開始。

3 精選限動證據庫（Highlight Proof Bank）

把 Stories 的高價值證據保存成 FAQ、案例、流程、工具、回饋。

4 個人檔案連結 / 私訊路徑（Link / DM Path）

讓訪客清楚知道低摩擦下一步，不把主頁變連結倉庫。

5 成效回收檢查（Performance Feedback）

用 profile visits、link clicks、DM、qualified DM 檢查漏點。

注意

Profile 做好不保證轉化，只能降低主頁承接斷點。

5. 個人簡介清晰度（Bio Clarity）

Bio 不是自我介紹，而是 Strategy 的壓縮版。

建議格式

我幫助 [具體受眾] 解決 [具體痛點 / 結果缺口] 用 [方法 / 系統 / 工具 / 經驗] 下一步：[低摩擦 CTA]

範例

幫 AI 新手建立工具學習路線 從工具清單 → 工作流 → 作品集 用 AK 系統把內容變成可執行流程 👇 領取 AI 學習路線檢查表

常見錯誤

  • 只寫頭銜。
  • 寫太多人生理念。
  • 沒有受眾。
  • 沒有下一步。
  • CTA 和內容 CTA 不一致。

6. 置頂內容信任矩陣（Pin Trust Matrix）｜置頂三件套

置頂內容不是放最紅影片，而是放最能讓冷流量理解你的三支內容。

Pin任務對應問題
Pin 1｜Who / Why你是誰，為什麼做這件事這個人和我有關嗎？
Pin 2｜Problem / Method你解決什麼問題，用什麼方法他能幫我解決什麼？
Pin 3｜Proof / Next Step證據、案例、入口我下一步去哪裡？

工具型 / 教學型替代版

Start Here Best Framework Free Resource / DM Entry

7. 精選限動證據庫（Highlight Proof Bank）

Stories 會消失，但 Highlight 可以保留信任證據。

Highlight任務
Start Here新訪客從哪裡開始。
FAQ常見問題與疑慮。
Results / Case案例與成果。
Process工作流程與方法。
Free Tool / Resource免費資源入口。
Behind幕後與真實過程。
Reviews學員 / 客戶回饋。

常見錯誤

Highlight 過多會變雜物櫃。保留 5–7 個核心 Highlight 即可，重點不是封面漂亮，而是能回答疑慮。

8. 個人檔案連結 / 私訊路徑（Link / DM Path）

Profile 的連結與 DM Trigger 要把受眾導向低摩擦下一步。

可用入口

  • DM 關鍵字
  • 免費檢查表
  • 模板
  • LINE / 社群
  • 預約諮詢
  • 低價產品
  • 工具頁
  • 等待名單

五個原則

  • 主入口只能有一個。
  • 次入口可以有，但不能搶焦點。
  • 入口要和 Strategy / Conversion Path 對齊。
  • 不要讓冷流量看到過多選項。
  • Link in Bio 是轉化分流器，不是連結倉庫。

9. Profile 成效回收檢查（Performance Feedback）

指標現象可能意義
profile visits 高、DM 低Bio / Pin / CTA 承接弱。
profile visits 高、link clicks 低連結入口不明或吸引力不足。
link clicks 高、DM 低landing / lead magnet / 表單斷點。
DM 多但品質差CTA 吸錯人或篩選不足。
profile visits 低Hook / Script 沒有足夠好奇心或信任。

真正要看的是這條鏈

Profile Visit → Link Click / DM → Qualified DM → Conversion

10. Profile Audit Checklist

個人簡介檢查（Bio Audit）

  • 是否明確說出服務誰？
  • 是否說出解決什麼問題？
  • 是否有方法 / 系統 / 角度？
  • 是否有單一下一步？
  • 是否和 Strategy 一致？

置頂內容檢查（Pin Audit）

  • Pin 1 是否回答你是誰？
  • Pin 2 是否回答你怎麼解？
  • Pin 3 是否回答證據 / 下一步？
  • 三支置頂是否互補？
  • 是否和 Conversion Path 一致？

精選限動檢查（Highlight Audit）

  • 是否有 Start Here？
  • 是否有 FAQ？
  • 是否有 Proof / Case？
  • 是否有 Process？
  • 是否刪除過期 Highlight？

連結 / 私訊路徑檢查（Link / DM Audit）

  • 是否只有一個主入口？
  • 主入口是否和內容 CTA 一致？
  • 是否有低摩擦入口？
  • 是否避免過多選項？
  • DM 第一問是否和入口一致？

11. Profile Failure Diagnosis

失敗症狀根因修正
主頁訪問高但沒 DM很多人點進來但不行動Bio / Pin 沒有下一步重寫 Bio + Pin 3。
主頁訪問高但 link clicks 低沒人點連結入口不明或選項太多單一主入口。
DM 多但品質差來的人不適合Bio / CTA 沒篩選加入 Not For / 篩選問題。
置頂觀看高但不轉化置頂很熱但無承接Pin 不是 Trust Matrix重新分配 Pin 1 / 2 / 3。
Highlight 很多但無用看起來像雜物櫃沒有 Proof Bank 架構重整成 5–7 個核心 Highlight。
Profile 和內容斷裂點進來像不同人Strategy / CTA / Stories 不一致統一主線與入口。

13. Unsupported Claims to Avoid

Profile 做好就一定成交

錯。Profile 只降低承接斷點，不保證成交。

Bio 寫對就一定漲粉

錯。Bio 是承接工具，不是流量保證。

置頂三件套會平台加權

錯。這是信任承接設計，不是 ranking factor。

連結越多越完整

錯。過多選項會提高決策摩擦。

14. 最終位置

Profile Architecture 是必要補強模組，不是新主線章節。正式位置建議：

Strategy → Profile Architecture → Signal → Hook → Script → CTA → Stories → Conversion → Algorithm → Performance → Execution
